# NEIS 종합의견 자동입력 도우미 (크롬 확장프로그램)

엑셀·구글시트에서 복사한 **학기말 종합의견 / 과세특 / 행발** 등을
나이스(NEIS)에 **순서·번호·이름** 매칭으로 자동 입력해 주는 크롬 확장프로그램입니다.

기존 도구와의 차이:

- 별도 웹앱/윈도우 exe 없이 **크롬 확장만으로** 동작 (맥·크롬북 포함)
- **템플릿 시트 제공** + 채운 파일(.xlsx/.csv) **업로드** → 자동 파싱
- 엑셀/구글시트에서 **바로 붙여넣기**도 지원
- **번호 / 이름** 매칭 또는 **순서대로** 입력(업로드 시 자동 판별)
- **이어쓰기**: 기존 평어 뒤에 한 칸 띄우고(또는 줄바꿈) 다른 평어 추가 — 덮어쓰지 않음
- 나이스 4세대(eXBuilder6) 그리드에 **API로 직접 입력**(가상스크롤 무관), 컬럼 자동 감지
- **되돌리기**, 입력 전 **미리보기·필드 스캔** 지원

> ⚠️ 나이스 공식 도구가 아닙니다. 개인정보는 수집하지 않으며 모든 처리는 브라우저 안에서만 이루어집니다.
> 자동입력 후에는 **반드시 값을 눈으로 확인하고 나이스에서 직접 저장**하세요.

## 설치 (개발자 모드 · 압축해제된 확장)

1. 크롬 주소창에 `chrome://extensions` 입력
2. 오른쪽 위 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램을 로드합니다** 클릭
4. 이 폴더(`neis-autofill`) 선택
5. 툴바에 확장 **아이콘**이 생기면 클릭 → 오른쪽에 사이드패널이 열립니다

`file://` 테스트 페이지를 쓰려면, 확장 상세 화면에서 **파일 URL에 대한 액세스 허용**을 켜세요.

## 사용법

1. **연습**: 먼저 `test/roster.html` 을 크롬으로 열어 연습하세요.
2. 데이터 준비 — 아래 둘 중 하나:
   - **(권장) 템플릿 방식**: 사이드패널의 **📄 템플릿 시트 다운로드** 클릭 → `번호/이름/학기말 종합의견`
     열을 채워 저장 → **파일 업로드**에 올리면 열 구성을 보고 입력 방식이 **자동 설정**됩니다.
     (구글시트는 파일>다운로드>.xlsx 또는 .csv 로 받아 업로드)
     > 💡 나이스에서 **[학기말 종합의견] 탭을 열고 [조회] 버튼을 누른 상태로** 템플릿을 다운로드하면, 그 화면의
     > **번호·이름이 템플릿에 미리 채워져** 내려받아집니다(전입/전출 괄호 표시는 빼고 이름만,
     > 결번은 결번대로 유지). 종합의견 칸만 채워 다시 올리면 이름으로 정확히 매칭됩니다.
   - **붙여넣기 방식**: 엑셀/구글시트에서 복사해 붙여넣기. 형식 예:
     `순서대로(내용만)`, `번호[탭]내용`, `이름[탭]내용`, `번호[탭]이름[탭]내용`
3. **① 미리보기** 로 파싱·바이트 확인
4. 나이스(또는 테스트) 종합의견 화면을 띄운 뒤 **② 필드 스캔** 으로 입력칸이 잡히는지 확인
5. (순서 방식이면) **첫 학생 입력칸을 클릭**한 뒤 **③ 자동입력 시작**
6. 값 확인 후 나이스에서 저장. 잘못되면 **되돌리기**

## 매칭 방식 선택 가이드

| 상황 | 추천 방식 |
| --- | --- |
| 시트가 명렬표 번호순으로 정렬됨 | **순서대로** (가장 안전) |
| 시트에 번호 열이 있음 | **번호** |
| 시트에 이름 열이 있음 | **이름** |

가상 스크롤(화면에 일부만 렌더)로 긴 명단일 때는 **순서대로** 방식이 한 칸씩
이동하며 입력하므로 가장 잘 동작합니다.

## 나이스 실제 화면에서 안 맞을 때 (튜닝 포인트)

나이스 4세대는 화면마다 DOM 구조가 다릅니다. 아래를 조정하세요.

- **필드가 안 잡힘**: `content.js` 의 `collectFields()` 셀렉터에 실제 입력칸 태그를
  추가하세요. (나이스가 커스텀 컴포넌트를 쓰면 `[contenteditable]` 이나 특정 class 필요)
- **번호/이름이 안 읽힘**: `parseIdentifier()` 의 정규식/셀 범위(depth)를 조정하세요.
- **입력은 됐는데 저장 시 값이 사라짐**: `setNativeValue()` 의 이벤트 목록에 해당
  프레임워크가 필요로 하는 이벤트(예: `compositionend`)를 추가하세요.

실제 나이스 종합의견 화면의 입력칸 HTML 한 조각(F12 → Elements)을 주면,
셀렉터와 매칭 로직을 그 구조에 맞게 정확히 고정할 수 있습니다.

## 파일 구성

```
neis-autofill/
├─ manifest.json      확장 설정 (MV3)
├─ background.js      사이드패널 열기
├─ sidepanel.html/css/js   붙여넣기·파싱·실행 UI
├─ content.js        나이스 페이지에서 필드 탐지·값 주입·되돌리기
├─ content.css
├─ lib/xlsx.full.min.js   엑셀 파싱 라이브러리(SheetJS, 로컬 동봉)
├─ template/         제공 템플릿 (.xlsx / .csv)
├─ test/roster.html  연습용 모의 명렬표
└─ icons/            아이콘
```
