# 크롬 웹스토어 등록 자료 — NEIS 종합의견 자동입력 도우미

개발자 대시보드( https://chrome.google.com/webstore/devconsole )에 그대로 복사해 넣을 수 있도록 정리한 문서입니다.
최초 등록 시 개발자 등록비 **US$5**(일회성)가 필요합니다.

---

## 1. 기본 정보

| 항목 | 값 |
| --- | --- |
| 이름 | NEIS 종합의견 자동입력 도우미 |
| 버전 | 1.0.2 |
| 카테고리(제안) | 업무 효율성(Workflow & Planning) 또는 교육 |
| 언어 | 한국어 |
| 업로드 패키지 | `dist/neis-autofill-1.0.2.zip` |

## 2. 짧은 설명 (요약, 132자 이내)

> 엑셀·CSV 파일의 내용을 나이스 학기말종합의견·행동특성및종합의견·창의적체험활동 화면에 번호·이름으로 자동 입력합니다.

## 3. 단일 목적 서술 (Single Purpose — 심사 필수)

> 이 확장의 유일한 목적은, 교사가 엑셀·CSV 파일에 미리 작성한 내용을 나이스(NEIS)의
> 학기말종합의견·행동특성및종합의견·창의적체험활동 입력 화면의 해당 학생 칸에 자동으로 채워 넣어,
> 반복적인 복사·붙여넣기 수작업을 줄이는 것입니다.

## 4. 상세 설명 (스토어 게시용)

```
엑셀·CSV 파일에 정리한 내용을 나이스(NEIS) 학생부 입력 화면에 한 번에 자동으로 채워 넣는
교사용 도구입니다. 별도 프로그램 설치 없이 크롬 확장만으로 동작합니다.

■ 지원 화면
· 학기말종합의견
· 행동특성및종합의견
· 창의적체험활동(학생부자료기록) — 자율·자치활동 동아리활동 + 진로활동 2칸 동시 입력

■ 주요 기능
· 템플릿 자동 생성 — 나이스 대상 화면에서 [조회] 후 이 버튼을 누르면 그 반의
  번호·이름이 채워진 템플릿이 내려받아집니다. 내용만 입력해 다시 올리면 됩니다.
  (창의적체험활동은 자율·자치활동·진로활동 2칸 템플릿으로 받아집니다.)
· 엑셀·CSV 파일 업로드로 간편 입력
· 번호·이름 매칭 또는 순서대로 자동 입력 (업로드 시 자동 판별)
· 이어쓰기 — 기존 내용을 덮지 않고 뒤에 이어서 추가
· 입력 전 미리보기, 입력 후 되돌리기

■ 개인정보 보호
모든 처리는 브라우저 안에서만 이루어지며, 어떤 데이터도 수집하거나 외부로 전송하지 않습니다.

■ 사용 안내
· 나이스 공식 도구가 아닙니다.
· 입력 전 2~3명으로 먼저 테스트하고, 값을 눈으로 확인한 뒤 나이스에서 직접 저장하세요.
```

## 5. 권한 사용 사유 (심사 폼의 Permission justification 에 입력)

| 권한 | 사유(그대로 입력) |
| --- | --- |
| `activeTab` | 사용자가 확장 아이콘을 눌러 실행한 현재 나이스 탭에 한해 입력 동작을 수행하기 위해 사용합니다. 사용자의 명시적 클릭이 있을 때만 해당 탭에 접근합니다. |
| `scripting` | 나이스 입력 화면(그리드·입력칸)에 사용자가 준비한 내용을 채워 넣는 스크립트를 프로그램적으로 주입하기 위해 사용합니다. |
| `sidePanel` | 데이터 준비(파일 업로드)와 실행 버튼 등 사용자 인터페이스를 브라우저 사이드패널로 표시하기 위해 사용합니다. |
| 호스트 권한 `*.neis.go.kr`, `*.eduptl.kr`, `*.edunavi.kr` | 확장이 나이스(*.neis.go.kr)와 시도교육청 업무포털(*.eduptl.kr, *.edunavi.kr) 화면에서만 동작하도록 대상을 제한하기 위해 사용합니다. 그 외 사이트에는 접근하지 않습니다. |

> 원격 코드(remote code)를 사용하지 않습니다. 엑셀 파싱 라이브러리(SheetJS)는 패키지에 로컬 동봉되어 있습니다.

## 8. 업로드 패키지 재생성 방법

소스 수정 후 배포본을 다시 만들려면 프로젝트 루트에서:

```
node build.js && (cd dist/neis-autofill && zip -rq ../neis-autofill-1.0.2.zip . -x '.*')
```

`build.js` 가 연습 페이지·심화진단·file:// 매칭을 제거한 클린 빌드를 `dist/` 에 생성합니다.
